# Řešení problémů

## Odhlašování uživatelů

Pokud je ve stránce mnoho 404 chyb, tak WebJET zapne ochranu a pro danou IP adresu nastaví nízký limit `session timeout` (ochrana před `DirBuster` / slovníkovými útoky). Limit lze zvětšit nastavením konfigurační proměnné `spamProtectionHourlyLimit-404`, ale lepší je vyřešit problém s chybějícími soubory/obrázky na stránce. Pokud tento problém nastává, tak se dá podívat v modulu Mazání dat / Cache objekty a hledat tam proměnnou `404-viewCount-IP-ADRESA` a prohlédnout si kolik je tam započtených pokusů. Pokud hodnota přesáhne 150 (což je minimální hodnota pro kterou se počet kontroluje), tak dochází k nastavení nízké session. Mírně odlišné je chování pro doménové jméno obsahující v názvu `cms`, tedy například pokud používáte pro správu stránky doménu `cms.domain.com`. V takovém případě není `session` okamžitě zrušena, ale je nastaven `session timeout` na 300 sekund. Ochranu lze také vypnout nastavením IP adresy do konfigurační proměnné `spamProtectionDisabledIPs`, to má ale i další dopady na vypnutí jiných ochran proti SPAMu. Přidáním IP adresy do tohoto seznamu umíte ale snadno diagnostikovat, zda se jedná o tento problém. Tato ochrana je také vypnuta pro administrátory.

Jiná možnost kdy toto nastává je, že se uživateli přepíná IP adresa, WebJET kontroluje změnu IP adresy `session` jako ochranu před ukradením session. Může k tomu dojít, pokud se používá proxy server s více IP adresami, nebo se uživateli přepíná WiFi a kabelové připojení. Lze nastavit konfigurační proměnná `sessionStealingCheck` na hodnotu `false` pro vypnutí této ochrany, ale snižuje se tak bezpečnost systému.
