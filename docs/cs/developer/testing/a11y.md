# Testování přístupnosti

Testování přístupnosti (accessibility, skr. **a11y**) ověřuje, zda jsou webové stránky a admin rozhraní použitelné pro všechny uživatele včetně těch se zdravotními omezeními. Testy se provádějí automatizovaně pomocí knihovny `axe-core` integrované do CodeceptJS přes helper `A11yHelper`.

## Knihovny a závislosti

| Knihovna | Účel |
| --- | --- |
| [axe-playwright](https://github.com/abhinaba-ghosh/axe-playwright) | Integrace `axe-core` do Playwright |
| [axe-core](https://github.com/dequelabs/axe-core) | Jádro pro detekci WCAG porušení |
| [axe-html-reporter](https://github.com/lpelypenko/axe-html-reporter) | Generování HTML reportu o porušeních |

## Úrovně WCAG

Testy kontrolují následující WCAG tagy (nakonfigurovány v `src/test/webapp/helpers/a11yhelper.js`):

| Tag | Popis |
| --- | --- |
| `wcag2a` | WCAG 2.0 úroveň A |
| `wcag2aa` | WCAG 2.0 úroveň AA |
| `wcag21a` | WCAG 2.1 úroveň A |
| `wcag21aa` | WCAG 2.1 úroveň AA |
| `wcag22aa` | WCAG 2.2 úroveň AA |

## Konfigurace

`A11yHelper` je nakonfigurován v `src/test/webapp/codecept.conf.js`:

```js
A11yHelper: {
    require: './helpers/a11yhelper.js',
    outputDir: '../../../build/test/allure-results',
    skipFailures: false,
    reporter: "html"
}
```

Stránkový objekt `a11y` je zaregistrován jako:

```js
a11y: './pages/a11y.js',
```

## Struktura souborů

```txt
src/test/webapp/
├── a11y.sh                        # Shell skript pre spustenie testov a generovanie Allure reportu
├── helpers/
│   └── a11yhelper.js              # CodeceptJS helper – obaľuje `axe-playwright`
├── pages/
│   └── a11y.js                    # Page object – metóda check() pre každý scenár
└── tests/
    └── a11y/
        └── xxx.js                  # Test prístupnosti
```

## Spuštění testů

### Příkazem npm

```sh
cd src/test/webapp
npm run a11y
```

Skript `a11y` spustí všechny testy ze složky `tests/a11y/**/*.js` s nastavením:

### Shell skript s Allure reportem

```sh
cd src/test/webapp
./a11y.sh [parameter]
```

Skript akceptuje volitelný parametr:

| Parametr | Příkaz | Popis |
| --- | --- | --- |
| *(bez parametru)* | `npm run a11y` | Spustí všechny a11y testy |
| `current` | `npm run a11y:current` | Spustí pouze aktuálně označené testy |
| `pause` | `npm run a11y:pause` | Spustí testy s pozastavením (Playwright pause) |

Příklady:

```sh
./a11y.sh
./a11y.sh current
./a11y.sh pause
```

Skript provede:

1. Vymaže předchozí výsledky z `build/test/` a `build/test-results/`
2. Spustí příslušný `npm run` příkaz podle parametru
3. Zkopíruje `allure/categories.json` do výsledků
4. Vygeneruje Allure HTML report do `build/test-results/`
5. Otevře report na portu `3001`

## Psaní testů

Každý a11y test využívá metodu `a11y.check()`, která spustí `axe-core` analýzu aktuální stránky a porovná počet porušení s hodnotou 0.

```js
Feature('a11y.nazovfunkcie');

Before(({ I, login }) => {
    login('admin');
});

Scenario('popis scenára', async ({ I, a11y }) => {
    I.amOnPage('/admin/v9/vasa-stranka/');
    await a11y.check();
});
```

### Parametr `context`

Metoda `a11y.check()` akceptuje volitelný parametr `context`, kterým omezíte kontrolu jen na určitou část stránky:

```js
// Kontrola len obsahu konkrétneho elementu
await a11y.check("#editorComponent");

// Kontrola elementu vnoreného v iframe
await a11y.check("iframe.cke_dialog_ui_iframe #editorComponent");
```

Parametr je mezerou dělený řetězec CSS selektorů. `A11yHelper` je zpracuje postupně:

1. Vyhledá první selektor v aktuálním kontextu stránky.
2. Pokud je nalezen element `<iframe>`, přepne kontext `axe` do obsahu tohoto `frame` a pokračuje dalším selektorem.
3. Je-li nalezen element jiný typ (např. `div`), použije zbytek řetězce jako `context` pro `axe`.
4. Pokud je poslední selektor samotný `<iframe>`, `axe` zkontroluje celý obsah `frame` bez dalšího omezení.

Tato logika umožňuje procházet libovolně hluboko do vnořených `iframe`, například pro dialogová okna.

### Co metoda `a11y.check()` dělá

1. Spustí `I.runA11yCheck()` (z `A11yHelper`)
2. V případě porušení:
   - Vytvoří chybovou zprávu ve formátu `IMPACT: id - popis`
   - Přidá přílohu s podrobným reportem chyb
3. Porovná počet porušení s 0 pomocí `I.assertEqual` – test selže, pokud jsou nějaká porušení

### Ukázka výstupu při selhání

```txt
3 accessibility violations were detected:

- SERIOUS: html-has-lang - Ensure every HTML document has a lang attribute
- CRITICAL: label - Ensure every form element has a label
- CRITICAL: select-name - Ensure select element has an accessible name
```

## Demo stránka

Soubor `src/main/webapp/components/aceintegration/admin/a11y.html` slouží jako referenční přístupná stránka pro pozitivní test (neobsahuje žádné WCAG porušení):

- Má `skip to main content` odkaz pro klávesnicové uživatele
- Všechny obrázky mají `alt` atribut
- Nadpisy jsou hierarchicky správné (`h1`, `h2`)
- Navigace má `aria-label`
- Tlačítka jsou sémanticky správně označena
