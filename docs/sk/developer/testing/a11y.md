# Testovanie prístupnosti

Testovanie prístupnosti (accessibility, skr. **a11y**) overuje, či sú webové stránky a admin rozhranie použiteľné pre všetkých používateľov vrátane tých so zdravotnými obmedzeniami. Testy sa vykonávajú automatizovane pomocou knižnice `axe-core` integrovanej do CodeceptJS cez helper `A11yHelper`.

## Knižnice a závislosti

| Knižnica | Účel |
| --- | --- |
| [axe-playwright](https://github.com/abhinaba-ghosh/axe-playwright) | Integrácia `axe-core` do Playwright |
| [axe-core](https://github.com/dequelabs/axe-core) | Jadro pre detekciu WCAG porušení |
| [axe-html-reporter](https://github.com/lpelypenko/axe-html-reporter) | Generovanie HTML reportu o porušeniach |

## Úrovne WCAG

Testy kontrolujú nasledujúce WCAG tagy (nakonfigurované v `src/test/webapp/helpers/a11yhelper.js`):

| Tag | Popis |
| --- | --- |
| `wcag2a` | WCAG 2.0 úroveň A |
| `wcag2aa` | WCAG 2.0 úroveň AA |
| `wcag21a` | WCAG 2.1 úroveň A |
| `wcag21aa` | WCAG 2.1 úroveň AA |
| `wcag22aa` | WCAG 2.2 úroveň AA |

## Konfigurácia

`A11yHelper` je nakonfigurovaný v `src/test/webapp/codecept.conf.js`:

```js
A11yHelper: {
    require: './helpers/a11yhelper.js',
    outputDir: '../../../build/test/allure-results',
    skipFailures: false,
    reporter: "html"
}
```

Stránkový objekt `a11y` je zaregistrovaný ako:

```js
a11y: './pages/a11y.js',
```

## Štruktúra súborov

```txt
src/test/webapp/
├── a11y.sh                        # Shell skript pre spustenie testov a generovanie Allure reportu
├── helpers/
│   └── a11yhelper.js              # CodeceptJS helper – obaľuje `axe-playwright`
├── pages/
│   └── a11y.js                    # Page object – metóda check() pre každý scenár
└── tests/
    └── a11y/
        └── xxx.js                  # Test prístupnosti
```

## Spustenie testov

### Príkazom npm

```sh
cd src/test/webapp
npm run a11y
```

Skript `a11y` spustí všetky testy z priečinka `tests/a11y/**/*.js` s nastavením:

### Shell skript s Allure reportom

```sh
cd src/test/webapp
./a11y.sh [parameter]
```

Skript akceptuje voliteľný parameter:

| Parameter | Príkaz | Popis |
| --- | --- | --- |
| *(bez parametra)* | `npm run a11y` | Spustí všetky a11y testy |
| `current` | `npm run a11y:current` | Spustí len aktuálne označené testy |
| `pause` | `npm run a11y:pause` | Spustí testy s pozastavením (Playwright pause) |

Príklady:

```sh
./a11y.sh
./a11y.sh current
./a11y.sh pause
```

Skript vykoná:

1. Vymaže predchádzajúce výsledky z `build/test/` a `build/test-results/`
2. Spustí príslušný `npm run` príkaz podľa parametra
3. Skopíruje `allure/categories.json` do výsledkov
4. Vygeneruje Allure HTML report do `build/test-results/`
5. Otvorí report na porte `3001`

## Písanie testov

Každý a11y test využíva metódu `a11y.check()`, ktorá spustí `axe-core` analýzu aktuálnej stránky a porovná počet porušení s hodnotou 0.

```js
Feature('a11y.nazovfunkcie');

Before(({ I, login }) => {
    login('admin');
});

Scenario('popis scenára', async ({ I, a11y }) => {
    I.amOnPage('/admin/v9/vasa-stranka/');
    await a11y.check();
});
```

### Čo metóda `a11y.check()` robí

1. Spustí `I.runA11yCheck()` (z `A11yHelper`)
2. V prípade porušení:
   - Vytvorí chybovú správu vo formáte `IMPACT: id - popis`
   - Pridá prílohu s podrobným reportom chýb
3. Porovná počet porušení s 0 pomocou `I.assertEqual` – test zlyhá, ak sú nejaké porušenia

### Ukážka výstupu pri zlyhaní

```txt
3 accessibility violations were detected:

- SERIOUS: html-has-lang - Ensure every HTML document has a lang attribute
- CRITICAL: label - Ensure every form element has a label
- CRITICAL: select-name - Ensure select element has an accessible name
```

## Demo stránka

Súbor `src/main/webapp/components/aceintegration/admin/a11y.html` slúži ako referenčná prístupná stránka pre pozitívny test (neobsahuje žiadne WCAG porušenia):

- Má `skip to main content` odkaz pre klávesnicových používateľov
- Všetky obrázky majú `alt` atribút
- Nadpisy sú hierarchicky správne (`h1`, `h2`)
- Navigácia má `aria-label`
- Tlačidlá sú sémanticky správne označené
