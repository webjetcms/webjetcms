## Translations of texts

In tests, it may be necessary to perform an action on the text, which changes depending on the language of the page/administration. A helper [i18n](../../../../src/test/webapp/pages/i18n.js) has been created, which simplifies the translation of texts between languages. It is assumed that the text input is in English and the output is in the language set in `ENV` variable `CODECEPT_LNG`. The current language is obtained using `I.getConfLng()`.

The translations themselves are in the files for [Slovak](../../../../src/test/webapp/pages/i18n.sk.js) and [Czech](../../../../src/test/webapp/pages/i18n.cs.js). The translations do not have a tree structure, it is assumed that the same text is always translated the same way. However, when entering a larger number of translations, we recommend that you write in the comment which test file they come from.

If the translation should have a different meaning than the one already used, it is possible to use a suffix in the format `::append`. For the English language, this suffix is ​​removed, but is used as part of the key when searching for a translation. Example: `Gallery->Galérie` and `Gallery::fbrowser->Galéria`.

The following methods are available:

- `get(text)` - ​​translation of text from English to the set language
- `click(text, context, options)` - ​​replacement of call `I.click` with text translation
- `see(text, context)` - ​​replacement of call `I.see(text, context)` with text translation
- `waitForText(text, sec, context)` - ​​replacement of call `I.waitForText(text, sec, context)` with text translation

Usage examples:

```javascript
Scenario('filebrowser - toolbar + navbar screens', ({ I, DTE, Document, i18n }) => {
    I.clickCss('div[title="'+i18n.get('Icons view')+'"].elfinder-button');
    I.click( locate("a.nav-link").withText(i18n.get("Tools")) );
});

Scenario('filebrowser - search screens', ({ I, Document, i18n }) => {
    i18n.click("In Subfolders");
    i18n.waitForText("File indexing completed");
});
```
