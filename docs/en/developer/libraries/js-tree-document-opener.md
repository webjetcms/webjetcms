# 'JsTree Document Opener'

Expanding library

---
**Dependencies**

- [Tools](tools.md)
---

The library enables automated manipulation (opening) of tree nodes,
which is generated by the jsTree library and opening the document based on its ID in the dataTables library.

Extends class `AbstractJsTreeOpener`
```javascript
import AbstractJsTreeOpener from "../abstract-opener/abstract-js-tree-opener";

export class JsTreeDocumentOpener extends AbstractJsTreeOpener {}
```

---
## Creating an instance:

**WebJET** creates an instance in the file [app.js](https://github.com/webjetcms/webjetcms/blob/main/src/main/webapp/admin/v9/src/js/app.js)
```javascript
import JsTreeDocumentOpener from "./libs/js-tree-document-opener/js-tree-document-opener";

window.jsTreeDocumentOpener = new JsTreeDocumentOpener();
```
**Usage:**

### Initialization:

Ideally, we should perform initialization whenever jstree is initialized by calling the [init()](#init) method.
```javascript
window.jsTreeDocumentOpener.init();
```

Wo **WebJET** is initializing in the file [app-init.js](https://github.com/webjetcms/webjetcms/blob/main/src/main/webapp/admin/v9/src/js/app-init.js)
```javascript
window.jstree = somStromcek.jstree({
    'core': {
        'check_callback': function (operation, node, parent, position, more) {},
        'data': function (obj, callback) {
            window.jsTreeDocumentOpener.init();
        }
    }
});
```

The [init()](#init) method will provide:<a id="init-algo"></a>
- Parses the search query after the question mark from the browser URL using [Tools.getUrlQuery()](tools.md#geturlquery)
- Selects the key `docid` from the parsed object if we haven't changed it before using [idKeyName](#idkeyname)
- The value from `docid` is set using the setter [id](#id)
- Setter [id](#id) resets the class to the new state and validates the input:<a id="id-algo"></a>
    - If the input is invalid, a message about invalid input will be printed in the DEV environment and execution will be interrupted
        - The class switches to the `Not Ready & Iddle` state, waiting for the setter [id](#id) to be called
    - If the input is valid, a request is made to the API, whose default address can be changed using [apiUrl](#apiurl)
        - The received data is saved to the class so that it can be worked with
        - The class switches to the `Ready & Iddle` state and waits for a call to [next()](#next) or [id](#id)

---

### Opening tree nodes:

Opening tree nodes is done by calling the [next()](#next) method:
```javascript
window.jsTreeDocumentOpener.next();
```

Wo **WebJET** is calling in the file [app-init.js](https://github.com/webjetcms/webjetcms/blob/main/src/main/webapp/admin/v9/src/js/app-init.js)
1. After rendering the tree `/listener: loaded.jstree`
2. After opening (_open node_) the node `/listener: after_open.jstree`
3. After selecting (_select node_) the node `/listener: select_node.jstree`

---
!>**Warning:**
If we want to set [id](#id) externally, it is a good idea to use the [loaded()](#loaded) method before that,
in which we can call [next()](#next) and test [notFound](#notfound).
```javascript
/** @type {JsTreeDocumentOpener} jstdo */
const jstdo = window.jsTreeDocumentOpener.loaded((result, docId, selfClass) => {
    if (selfClass.notFound) {
        // Napr. nejaký alert pre používateľa
    } else {
        selfClass.next();
    }
});

jstdo.id = 4;
```

The problem with the implementation was waiting for the tree structure to load and the data table to synchronize. The original implementation used ```this.dataTable.on('draw.dt', (evt, settings) => ``` events, but this was not suitable for use when switching between the Folders, System and Recycle Bin tabs, because the data table was loaded immediately after switching the tab.

The code uses the function ```setInterval``` with the call counting in the function ```waitForDatatableRowLoaded``` and the waiting for the not to be displayed ```div.dt-processing:visible```. When it is not displayed, it counts 3 more intervals of not to be displayed and then continues searching for the web page. Loading the tree structure is solved similarly, the waiting is implemented in the function ```waitForJsTreeLoaded```. The functions are implemented directly in ```abstract-js-tree-opener.js```.

The search is also complicated due to pagination and the differences between client and server paging. In both cases, it sequentially traverses each page and looks for an element with the specified ID. If it is found, it opens the editor.

If the value ```docid``` is specified as the parameter ```-1```, a new editor will be opened by simulating a click on the Add button.

When combining the specified parameter ```groupid```, the following situations are possible:

- positive ```docid``` is specified - parameter ```groupid``` is ignored, the directory will be opened according to the directory of the specified web page
- ```docid=-1``` is entered, the parameter ```groupid``` is used to open the directory structure and then the opening of a new web page is invoked

### Detailed description of functions

#### init()

Invokes initialization that provides [this following procedure](#init-algo)
```javascript
/**
 * @description Inicializácia po načítaní jstree
 * @returns {void}
 * @public
 */
window.jsTreeDocumentOpener.init();
```
---
#### next()

Opens the next node in the sequence. If there is no other node in the list, it opens the dataTable document with our ID.
```javascript
/**
 * @description Otvorí nasledujúci node v poradí.
 *              Ak už neexistuje v zozname žiadny ďalší node, otvorý dataTable dokument s našim ID.
 * @returns {void}
 * @public
 */
window.jsTreeDocumentOpener.next();
```
---
#### loaded()

Sets a callback that will be executed after a new [id](#id) is entered and data is received.
```javascript
/**
 * @description Nastaví callback, ktorý sa vykoná po zadaní nového DocID a prijatí dát.
 * @param {Function} callback [result, docId, JsTreeDocumentOpener]
 * @returns {JsTreeDocumentOpener}
 * @public
 */
window.jsTreeDocumentOpener.loaded(callback);
```
---
#### inputDataFrom()

Appends the specified input, which must be inserted as a jQuery object `$('.input-css-trieda')`.
Input ensures the entry of a new ID
```javascript
/**
 * @description Pripojí zadaný input, ktorý musí byť vložený ako jQuery objekt `$('.input-css-trieda')`
 *              Input zabezpečuje zadanie nového ID
 * @param {jQuery} openerInput jQuery selector for input
 * @param {function} [withNotifyCallback] Callback, ktorý sa vykoná, pri chybnom vstupe alebo nenájdenom dokumente.
 *                                        V callbacku je dostupný argument `value`
 * @returns {void}
 * @public
 */
window.jsTreeDocumentOpener.inputDataFrom(openerInput, withNotifyCallback);
```
---

#### setInputValue()

If it was connected using [inputDataFrom()](#inputdatafrom) input, we can set a value to that input from anywhere.
```javascript
/**
 * @description Ak bol pripojený pomocou inputDataFrom() input, tak danému inputu môžeme odkiaľkoľvek nastaviť value.
 * @see inputDataFrom
 * @param {string|number} value
 * @returns {void}
 * @public
 */
window.jsTreeDocumentOpener.setInputValue(value);
```
---
#### id

Setting a new document ID.
This ID will be sent via the API and will also be used to open the document after the relevant tree nodes have been successfully opened.

[Click for more info on what setter id does](#id-algo)
```javascript
/**
 * @description Nastavenie nového ID dokumentu.
 *              Toto ID bude odoslané cez API a zároveň bude použité na otvorenie dokumentu po úspešnom otvorení príslušných uzlov stromu.
 * @default docid
 * @param {number|string} id
 * @returns {void}
 * @public
 * @setter
 */
window.jsTreeDocumentOpener.docId = value;
```
---
#### apiUrl

Setting up a new API url
```javascript
/**
 * @description Nastavenie novej API url

 * @example
 *          API url bez ID
 *          `/admin/rest/web-pages/parents/`
 * @returns {void}
 * @public
 * @setter
 */
window.jsTreeDocumentOpener.apiUrl = value;
```
---
#### idKeyName

The new name of the URL query key that will contain the document ID.
```javascript
/**
 * @description Nový názov URl query kľúča, v ktorom sa bude nachádzať ID dokumentu.
 * @param {string} value
 * @returns {void}
 * @public
 * @setter
 */
window.jsTreeDocumentOpener.idKeyName = value;
```
---
#### notFound

It is used to determine whether, after receiving data, there is a node in the jsTree that we want to open first.
If such a node does not exist (not found), the value TRUE is returned.
If such a node exists (was found), the value FALSE is returned.

The ideal use of this getter is inside the [loaded()](#loaded) method.
```javascript
/**
 * @description Slúži na zistenie, či po prijatí dát, existuje v jsTree uzol, ktorý chceme otvoriť ako prvý.
 *              Ak takýto uzol neexistuje (nebol nájdený / not found), tak sa nám vráti hodnota TRUE
 *              Ak takýto uzol existuje (bol nájdený / found), tak sa nám vráti hodnota FALSE
 *              Ideálne použitie tohto gettera je vo vnútri metódy loaded()
 * @returns {boolean}
 * @public
 * @getter
 */
window.jsTreeDocumentOpener.notFound;
```
