# Datatable columns

The JSON "columns" array for DataTables can be generated by annotating the properties of a Java object. The JSON is inserted into the pug file by calling:

```javascript
let columns = [(${layout.getDataTableColumns('sk.iway.iwcm.components.gallery.GalleryEntity')})];
```

Where ```sk.iway.iwcm.components.gallery.GalleryEntity``` is an object managed via DataTables.

All object fields that have the annotation ```@DataTableColumn``` are mapped to JSON.

The annotation ```@DataTableColumn``` has the same properties as the [original columns object](../datatables/README.md). Furthermore, it is possible to set certain properties using the inputType shortcut, which are defined by ```enum dataTableColumnType```.

Editor properties are set using the `DataTableColumnEditorAttr` annotation. The value can be entered directly or retrieved from a configuration variable using the `constant:` prefix:

```java
    @DataTableColumn(inputType = DataTableColumnType.JSON, tab = "basic", className = "dt-tree-dir-simple", title="components.file_archiv.target_directory", hidden = true, editor = {
        @DataTableColumnEditor(
            attr = {
                @DataTableColumnEditorAttr(key = "data-dt-field-root", value = "constant:fileArchivDefaultDirPath"),
                @DataTableColumnEditorAttr(key = "data-dt-field-skipFolders", value = "fileArchivInsertLaterDirPath"),
            }
        )
    })
    private String dir;
```

## @DataTableColumn properties

Original documentation on [datatables.net](https://datatables.net/reference/option/columns:)

Required fields:

- ```inputType``` - ​​abbreviation, ```enum DataTableColumnType``` - specifies the type of data field. It is also possible to use a combined value (e.g.: ```inputType = { DataTableColumnType.OPEN_EDITOR, DataTableColumnType.JSON },```).
- ```title``` - ​​if not specified, it is automatically generated as ```components.meno_beanu_bez_dto_alebo_bean_na_konci.name``` - https://datatables.net/reference/option/columns.title, see the documentation for [translations](#column-name-translations). **Warning:** if title is empty (or a hard space), the ```hidden=true``` attribute is automatically set for the column.

Optional fields:

- ```tab``` - ​​shortcut for setting ```{editor: {tab: String}}```
- ```className``` - ​​additional CSS style set on ```TD``` element, if you want to replace the CSS style set with ```inputType``` enter it with ```!``` at the beginning - https://datatables.net/reference/option/columns.className. There are special CSS classes:
  - `disabled` - ​​sets the field to gray, which indicates that it is non-editable.
  - `DTE_Field_Has_Checkbox` - ​​sets the bottom indent to the value `-14px` so that there is no space before the next field.
  - `hide-on-create` - ​​hides the field when creating a new record.
  - `hide-on-edit` - ​​hides the field when editing a record.
  - `hide-on-duplicate` - ​​hides the field when duplicating a record.
  - `not-export` - ​​field will not be exported.
  - `show-html` - ​​the HTML code in the value is displayed, including entities of type `&#39;`, the column is set by the `entityDecode: false` attribute.
  - `wrap` - ​​turns on text wrapping, used primarily in fields of type `textarea`.
  - `multiweb-noteditable` - ​​in a MultiWeb installation, the field is displayed in gray, which indicates that it is non-editable.
- ```name``` - ​​https://datatables.net/reference/option/columns.name
- ```data``` - ​​https://datatables.net/reference/option/columns.data
- ```defaultContent``` - ​​https://datatables.net/reference/option/columns.defaultContent
- `orderable` - ​​`true/false` value for enabling sorting options in the data table
- ```renderFormat``` - ​​https://datatables.net/reference/option/columns.renderFormat
- ```renderFormatLinkTemplate``` - ​​https://datatables.net/reference/option/columns.renderFormatLinkTemplate
- ```renderFormatPrefix``` - ​​https://datatables.net/reference/option/columns.renderFormatPrefix
- `renderFunction` - ​​name of the JavaScript function that will be used for custom rendering of the column value. More on the [DataTables page](https://datatables.net/reference/option/columns.render).
- `sortAfter` - ​​field name after which this field will be added in the sequence
- ```editor``` - ​​object ```DataTableColumnEditor```
- ```hidden``` - ​​the field will not be displayed in the data table and the user cannot view it, unlike ```visible```, the field can be used in the editor
- ```hiddenEditor``` - ​​if ```true``` the column will not be displayed in the editor
- ```visible``` - ​​the field will be hidden, but the user can view it https://datatables.net/reference/option/columns.visible
- ```filter``` - ​​if false, the filter will not be displayed in the table header
- ```perms``` - ​​value for rights control (e.g. multiDomain), the field will not be displayed unless the logged in user has the specified right enabled
- ```defaultValue``` - ​​default value for a new record (used only if ```fetchOnCreate``` is set to ```false```, because in this case the default data from the server will be returned). May contain macros:
  - ```{currentDomain}``` - ​​will be replaced by the currently selected domain
  - ```{currentDate}``` - ​​will be replaced with the current date
  - ```{currentDateTimeSeconds}``` - ​​will be replaced with the current date and time including seconds
  - ```{currentTime}``` - ​​will be replaced with the current time
- `alwaysCopyProperties` - ​​when editing a record, empty `null` values ​​are preserved and copied from the existing object in the database. This does not apply to date/time fields, which are automatically overwritten. If you need to use this for another field type and also transfer the `null` value, set the attribute to `true`, or to `false` if you do not want automatic overwriting for date fields.
- `ai` - ​​by setting it to the value `false` it is possible to disable the display of the AI ​​icon for general options (translate, correct grammar...). The AI ​​icon will only be displayed if the assistant is set for this specific field.
- `disabled` - ​​setting it to `false` sets the attribute `disabled="disabled"` to the input field in the editor.
- `export` - ​​setting to the value `false` will not export the field.

## @DataTableColumnEditor properties

- ```type``` - ​​type of ```input``` element
- ```label``` - ​​translation key of the field name in the editor (if different from ```DatatableColumn.title```)
- ```message``` - ​​translation key for displaying the tooltip, if not specified, a translation for the key is automatically searched for according to ```DatatableColumn.title.tooltip```. Supports specifying formatting using basic [Markdown](../frameworks/webjetjs.md#markdown-parser).
- ```tab``` - ​​[tab in which the field is located](README.md#tabs-in-editor)
- ```attr``` - ​​```HashMap``` HTML attributes that will be set to the input field
  - ```data-dt-field-hr``` (```before/after```) - adds a dividing line before or after an element
  - ```data-dt-field-headline``` (translation key) - adds a title before the element
  - ```data-dt-field-full-headline``` (translation key) - adds a full-width title before the element (including the gray area with element names), used for a title before a nested data table in a separate tab
- ```multiple``` - ​​sets the ```multiple``` attribute on the HTML field (used for a field of type ```MULTISELECT```).
- ```separator``` - ​​sets the separator character for the field of type ```MULTISELECT```. If empty, data is sent and received as a field, if set as a string separated by the specified character (typically a comma).
- `data-dt-escape-slash` - ​​setting it to the value `true` will enable the replacement of the character `/` with the entity `&#47;`. It is used in the case of a web page and a folder where the character `/` needs to be replaced in the name, as it is used to separate the path.

The translation key for the tooltip is automatically searched for by the translation key ```title``` with the suffix ```.tooltip```. So if you have an annotation ```@DataTableColumn(title = "group.superior_directory"```, the translation text with the key ```group.superior_directory.tooltip``` is automatically searched for. If it exists, it is used.

## DataTableColumnType properties

Sets the field type, more in the list of [standard form fields](standard-fields.md).

- ```ID``` - ​​primary key column
- ```OPEN_EDITOR``` - ​​automatically creates a link to open the editor on the column, should be used for the main text field, ideally the first one in the sequence
- ```TEXT``` - ​​standard text field (one line)
- ```TEXTAREA``` - ​​standard field for entering multiple lines of text
- ```SELECT``` - ​​selection field, we recommend sending options via [REST service](../datatables/restcontroller.md#dials-for-select-boxes)
- ```MULTISELECT``` - ​​multiple choice selection field
- ```BOOLEAN``` - ​​checkbox with options ```true/false```
- ```CHECKBOX``` - ​​checkbox with special value, option for selected and unselected value can be set with editor attribute ```@DataTableColumnEditorAttr(key = "unselectedValue", value = "")```
- ```DISABLED``` - ​​the displayed field will not be editable

Numeric:

- ```NUMBER``` - ​​numeric field
- ```TEXT_NUMBER``` - ​​displays a rounded number, for higher numbers it displays in text form, e.g. ```10 tis.``` instead of ```10000```
- ```TEXT_NUMBER_INVISIBLE``` - ​​numeric field that will not be displayed in the table or in the editor

Date:

- ```DATE``` - ​​field for entering a date, clicking in the field displays a calendar selection
- ```DATETIME``` - ​​field for entering date and time, after clicking in the field a calendar selection will be displayed with the option to enter time data
- ```TIME_HM``` - ​​field for entering time only, after clicking in the field a selection will appear with the option to select hours and minutes
- ```TIME_HMS``` - ​​field for entering time only, after clicking in the field a selection will appear with the option to select hours, minutes and seconds
- `DURATION` - ​​field for displaying duration in `mm:ss` format, input is `long` value in milliseconds

Special:

- ```GALLERY_IMAGE``` - ​​special field type for displaying an image with the column title disabled
- ```QUILL``` - ​​simple editor with basic HTML code formatting like bold, italics, etc.
- ```WYSIWYG``` - ​​full-featured HTML code editor for a website
- ```JSON``` - ​​field for selecting [directory](field-json.md)
- ```DATATABLE``` - ​​[nested datatable](field-datatable.md)
- ```ELFINDER``` - ​​[link selection](field-elfinder.md) to file / web page
- ```UPLOAD``` - ​​[file upload](field-file-upload.md)

## Selection field options

You can set ```DataTableColumnType.SELECT``` fields to ```option``` values ​​via:

- [REST service](../datatables/restcontroller.md#Select-box-codes) and setting up select box code lists. This is the preferred solution for standard datatables.
- By setting the options attributes directly using the ```@DataTableColumnEditorAttr(key = "Slovensky", value = "sk")``` annotation.
- By calling a static method using the ```@DataTableColumnEditorAttr(key = "method:sk.iway.basecms.contact.ContactRestController.getCountries", value = "label:value")``` annotation. In the ```key``` attribute, the class prefixed with ```method:``` and the method that must return ```List``` objects are specified. In the ```value = "label:value"``` attribute of the annotation, the attribute name for the description and the attribute name for the value of the selection field are specified (in the example, it is called ```objekt.getLabel() a objekt.getValue()```).
- By adding the annotation `@DataTableColumnEditor.optionMethods` with the specified method to execute (this is a nicer notation compared to the previous method). The value of `labelProperty` and `valueProperty` is used to get the text and the value of `option` of the element, if not specified it is obtained from the `label` and `value` attributes.
- By connecting to the codebook application by entering ```@DataTableColumnEditorAttr(key = "enumeration:Okresne Mestá", value = "string1:string2")```. In the ```key``` attribute, the prefix ```enumeration:``` is entered, the name or ID of the codebook. In the ```value = "string1:string2"``` attribute of the annotation, the name of the attribute for the description and the name of the attribute for the value of the selection field are entered - in the example it is called ```objekt.getString1() a objekt.getString2()```.

```java
@DataTableColumn(inputType = DataTableColumnType.SELECT, tab = "basic", editor = {
        @DataTableColumnEditor(
                options = {
                        //klasicky option tag
                        @DataTableColumnEditorAttr(key = "Slovensky", value = "sk"),
                        @DataTableColumnEditorAttr(key = "Česky", value = "cz"),

                        //ukazka ziskania zoznamu krajin volanim statickej metody, vo value su mena property pre text a hodnotu option pola
                        @DataTableColumnEditorAttr(key = "method:sk.iway.basecms.contact.ContactRestController.getCountries", value = "label:value"),

                        //ukazka napojenia na ciselnik, mozne je zadat meno alebo ID ciselnika, vo value su mena property pre text a hodnotu option pola
                        @DataTableColumnEditorAttr(key = "enumeration:Okresne Mestá", value = "string1:string2")
                }
        )
        @DataTableColumn(inputType = DataTableColumnType.SELECT, title = "components.formsimple.fieldType", editor = {
        @DataTableColumnEditor(
                optionMethods = {
                        @DataTableOptionMethod(
                                className = "sk.iway.iwcm.components.formsimple.FormSimpleApp",
                                methodName = "getFieldTypes",
                                labelProperty = "label",
                                valueProperty = "value"
                        )
                })
        })
        private String fieldType;
})
private String country;
```

## Required fields

Required fields can be marked with annotations:

- ```@NotEmpty``` - ​​does not empty the field, does not allow entering a space or tab
- ```@NotBlank``` - ​​does not empty the field, but allows you to enter a space

Other validation options are described in the [restcontroller] documentation (../datatables/restcontroller.md#validation--required-fields).

## JS Helper for property extension

The file ```app-init.js``` contains the function ```WJ.DataTable.mergeColumns``` for adding properties based on the object name (field name).
Iterates over the columns array of objects, finds an object with the same name as the object argument, and extends this object using ```jQuery.extend```.

```javascript
WJ.DataTable.mergeColumns(columns, {
    name: "datatableImage",
    render: function (data, type, row) {
        return '<div class="img" style="background-image:url(/thumb' + row.imagePath + '/' + row.imageName + '?w=600&h=400&q=90&v=' + (new Date()).getTime() + ');"></div>';
    },
    className: "dt-image",
    renderFormat: "dt-format-none"
});
```

If you just need to add a new column, you can do it simply by adding it to the list. In the example, we are adding a *hidden* field (attribute ```visible: false```):

```javascript
columns.push({
        data : "availableGroups",
        name : "availableGroups",
        title : "[[\#{admin.temp.edit.showForDir}]]",
        visible: false,
        editor: {
                type : "text",
                tab : "accessTab"
        }
});
```

If you need to change the order of columns, you can do so using the ```WJ.DataTable.moveColumn``` function. In the example, the column named ```formSettings.recipients``` is moved after the column named ```formName```:

```javascript
filteredColumns = window.WJ.DataTable.moveColumn(filteredColumns, "formSettings.recipients", "formName");
```

## Nested attributes

It is often necessary to add additional attributes to the entity for the editor (e.g. ```checkbox``` to apply the change to subordinate entities, an additional field with information, etc.). For this purpose, the entity can be extended with a new attribute (which is not stored in the database) containing additional data. Typically, we call it ```editorFields``` and implement the necessary class for the entity. Examples are in [DocEditorFields](../../../../src/main/java/sk/iway/iwcm/doc/DocEditorFields.java) or [GroupEditorFields](../../../../src/main/java/sk/iway/iwcm/doc/GroupEditorField.java). In the classes, there is then only the editorField attribute, e.g. ```private DocEditorFields editorFields = null;```.

An implemented class ```EditorFields```, e.g. [DocEditorFields](../../../../src/main/java/sk/iway/iwcm/doc/DocEditorFields.java) typically contains methods ```fromDocDetails``` for setting attributes in the ```editorFields``` class before editing and ```toDocDetails``` for resetting attributes in ```DocDetails``` before saving. These methods need to be called implicitly in your Java code.

!>**Warning:** if the entity is cached (such as [GroupDetails](../../../../src/main/java/sk/iway/iwcm/doc/GroupDetails.java)) the ```editorFields``` attribute setting will remain in the cache and may unnecessarily consume memory and create unnecessarily large data during JSON serialization. In ```GroupDetails``` in editorFields it refers to ```parentGroupDetails```.

In the standard procedure, each ```GroupDetails``` object was set to a ```editorFields``` object. When serializing a deeply nested directory, the editorFields.parentGroupDetails.editorFields.parentGroupDetails objects were subsequently nested. The GroupDetails object did not have the necessary first editorFields. The solution is to first clone the ```GroupDetails``` object and then set ```editorFields``` to it. An example is in ```GroupEditorField.fromGroupDetails``` which clones the object and then returns it. The usage in the code is then as ```group = gef.fromGroupDetails(group);```.

Common methods for a datatable are in the [BaseEditorFields](../../../../src/main/java/sk/iway/iwcm/system/datatable/BaseEditorFields.java) class, which your class can extend. It contains methods for adding a CSS row class and adding an icon to the title. More in the [datatable styling](../datatables/README.md#row-styling) documentation.

To insert an annotation for nested attributes, you can use the annotation ```@DatatableColumnNested```, such as in [DocDetails](../../../../src/main/java/sk/iway/iwcm/doc/DocDetails.java) on the attribute ```editorFields```:

```java
@DataTableColumnNested
@Transient
private DocEditorFields editorFields = null;
```

the attribute annotated in this way will be searched for the annotation ```@DatatableColumn``` recursively. The result will be the generation of a nested annotation. Notice the data and name attributes containing the prefix ```editorFields.```. If you need to set your own prefix (variable name), you can use the prefix parameter ```@DataTableColumnNested(prefix = "menoPremennej")```. You can also set the prefix to an empty value, in which case the prefix will not be generated.

```javascript
{
  "data" : "editorFields.allowChangeUrl",
  "name" : "editorFields.allowChangeUrl",
  "title" : "Povolit zmenu URL",
  "renderFormat" : "dt-format-boolean-true",
  "editor" : {
    "type" : "checkbox",
    "tab" : "basic"
  }
}, {
  "data" : "editorFields.test",
  "name" : "editorFields.test",
  "title" : "TeST hodNOTa",
  "renderFormat" : "dt-format-text",
  "editor" : {
    "type" : "text",
    "tab" : "basic"
  }
}
```

The ```@Transient``` annotation tells JPA entities that the given attribute is not stored in the database.

To set data between an entity and ```editorFields``` in a REST controller, you can implement methods ```processFromEntity``` to set ```editorFields``` attributes or ```processToEntity``` to set attributes in an entity from ```editorFields```. An example can be seen in [UserDetailsController](../../../../src/main/java/sk/iway/iwcm/components/users/userdetail/UserDetailsController.java). The methods are automatically called when reading all records, retrieving a single record, searching, or saving data.

```java
    /**
         * Vykona upravy v entite pred vratenim cez REST rozhranie
         * napr. vyvola potrebne editorFields nastavenia (from entity to editorFields)
         * @param entity
         * @param action - typ zmeny - create,edit,getall...
         */
    @Override
    public UserDetailsEntity processFromEntity(UserDetailsEntity entity, ProcessItemAction action) {
        boolean loadSubQueries = false;
        if (ProcessItemAction.GETONE.equals(action)) {
            loadSubQueries = true;
            if (entity == null) entity = new UserDetailsEntity();
        }

        if(entity != null && entity.getEditorFields() == null) {
            UserDetailsEditorFields udef = new UserDetailsEditorFields();
            udef.fromUserDetailsEntity(entity, loadSubQueries, getRequest());
        }
        return entity;
    }

    /**
	 * Vykona upravy v entite pri zapise cez REST rozhranie
	 * napr. vyvola potrebne editorFields nastavenia (from editorFields to entity)
	 * @param entity
	 * @param action - typ zmeny - create,edit,getall,
	 */
    @Override
    public UserDetailsEntity processToEntity(UserDetailsEntity entity, ProcessItemAction action) {
        if(entity != null) {
            //Call toUserDetailsEntity to set new entity values from EditorFields
            UserDetailsEditorFields udef = new UserDetailsEditorFields();
            udef.toUserDetailsEntity(entity, getRequest());
        }
        return entity;
    }
```

If we have another entity as a nested attribute, it is a good idea to also set the `sortPrefix` attribute to the value of the attribute name. This prefix will automatically be added to the `sortAfter` field on the attributes of the nested entity, otherwise you will get mixed up field names from different entities.

```java
public class FormsEntityBasic {

    // Numeric value of the same column (user_id)
    @Column(name = "user_id")
    private Long userId;

    // Relation to users table; load lazily, only readable
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id", insertable = false, updatable = false)
    @JsonProperty(access = JsonProperty.Access.READ_ONLY)
    @DataTableColumnNested(sortPrefix = "userDetails.")
    private UserDetailsEntity userDetails;

}
```

## Sorting the order of fields

By default, fields are ordered in the order they are written in the source code (although the annotation specification does not guarantee this, it works that way). However, if you use nested attributes, you cannot set the order in the code.

Therefore, it is possible to use the ```sortAfter``` attribute, in which you enter the data attribute of the previous field. The annotated field is then added to the JSON output after the specified field. The logic is implemented in the [DataTableColumnsFactory.sortColumns](../../../../src/main/java/sk/iway/iwcm/system/datatable/DataTableColumnsFactory.java) method.

If necessary, a special value `sortAfter = "FIRST"` can be specified to move the field to the beginning of the list. This is necessary for extended entities via `@MappedSuperclass` if the first `id` attribute is also in this entity.

Example of use:

```java
    @DataTableColumn(inputType = DataTableColumnType.BOOLEAN, title = "Povolit zmenu URL", tab = "basic",
        sortAfter = "virtualPath"
    )
    private Boolean allowChangeUrl=false;
```

## Column name translations

As mentioned above, the ```title``` attribute contains the column name. Due to translations, you cannot use the Slovak column name directly, you need to use a translation key.

If you do not specify the ```title``` attribute, a translation key in the format ```components.meno_beanu_bez_dto_alebo_bean_na_konci.name``` is automatically searched for, e.g. ```components.monitoring.date_insert```.

**If you are modifying an existing application/component** from WebJET 8 to WebJET 2021, it is best to find the original translation keys. This will save time when translating, as WeBJET 8 is already translated into multiple languages.

The original translation file from WebJET 8 is in the file [src/main/webapp/files/text.properties](../../../../src/main/webapp/WEB-INF/classes/text.properties) (for demonstration purposes only, do not modify it in any way). You can search for the desired text in it and enter the found translation key in the ```title``` attribute.

Another option is to display the original page with the URL parameter ```?showTextKeys=true``` which will cause the translation keys to be displayed before the text. The page will probably be broken from a design point of view (since the texts will be too long), but you can see the keys through the inspector.

For example:

```txt
http://iwcm.interway.sk/components/server_monitoring/admin_monitoring_all.jsp?showTextKeys=true
```

If you already submit the form on the page, of course you will see the original texts, you add the parameter as ```&showTextKeys=true```, but you will probably see a 403/404 page due to WebJET protection. The solution is to use the JavaScript console, where you enter:

```javascript
window.location.href=window.location.href+"&showTextKeys=true";
```

This will correctly pass WebJET's protection and the keys will be displayed to you.

**If you have created a new application, or you have not found a suitable translation key**, you need to add it to the [text-webjet9.properties](../../../../src/main/webapp/WEB-INF/classes/text-webjet9.properties) file.

After adding the translation, you need to reload the ```text-webjet9.properties``` file with WebJET. You can do this by calling the [homepage with the parameter ?userlngr=true](http://iwcm.interway.sk/admin/?userlngr=true) or by restarting the application server.
